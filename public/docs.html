Technical Architecture

A serverless, high-throughput pipeline engineered for massive scale. Leveraging GitHub Releases as a globally distributed CDN database.

Core Principles

Stateless Ingestion

Python workers on GitHub Actions perform ETL operations, eliminating the need for persistent intermediate servers.

Release-as-Database

Processed datasets are versioned and stored as immutable artifacts, serving as a high-speed read layer.

Proxy Layer

FastAPI acts as a thin authentication and redirection layer, ensuring PowerBI consumes optimized data streams.

Integration Points

GET /api/integration/powerbi

Primary Integration Stream. Redirects to the master dataset artifact. Optimized for PowerBI web ingestion.
